## Overview

App CR provides REST API endpoints for user and task management. This section documents all available endpoints, their parameters, and expected responses.

## Current Implementation Status

<Warning>
  **Limited Implementation**: Currently, only the user registration endpoint is implemented. Task endpoints are documented based on the database schema but require backend implementation.
</Warning>

### Implemented Endpoints

- ✅ **POST /usuarios** - Create a new user

### Planned Endpoints (Not Yet Implemented)

- ❌ **GET /usuarios** - Retrieve users
- ❌ **POST /tasks** - Create a task
- ❌ **GET /tasks** - Retrieve tasks
- ❌ **PUT /tasks/:id** - Update a task
- ❌ **DELETE /tasks/:id** - Delete a task

## Base URL

All API endpoints are relative to your server's base URL:

```text
http://localhost:3000
```

## Request/Response Format

All requests and responses use JSON format. Include the Content-Type header:

```bash
Content-Type: application/json
```

## Authentication

<Note>
  Authentication is not currently implemented. JWT tokens and bcrypt are installed but not yet integrated into the endpoints.
</Note>

When authentication is implemented, protected endpoints will require a JWT token in the Authorization header:

```bash
Authorization: Bearer YOUR_JWT_TOKEN
```
